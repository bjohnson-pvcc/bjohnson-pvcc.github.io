Chapter 7: Software Engineering

Term Definition
Software engineering The systematic approach to the design, development, testing, and maintenance of software.
Integrated development environment (IDE) A software application that provides comprehensive facilities to computer programmers for software development.
Association for Computing Machinery (ACM) A professional organization for computer scientists and IT professionals that aims to advance computing as a science and a profession.
Purpose of ACM To provide a platform for professionals to share knowledge and experiences, to provide education and training opportunities, and to promote ethical and professional practices in the computing industry.
Benefits of ACM membership Access to online resources, networking opportunities, discounts on conferences and publications, and career development support.
Student ACM membership A discounted membership option for students who are pursuing a degree in computing or IT-related fields.
Institute of Electrical and Electronics Engineers (IEEE) A professional organization for electrical, electronics, and computer engineers that aims to advance technology and innovation.
Purpose of IEEE To promote the advancement of technology and engineering, to provide education and training opportunities, and to establish standards for the computing industry.
Benefits of IEEE membership Access to online resources, networking opportunities, discounts on conferences and publications, and career development support.
Student IEEE membership A discounted membership option for students who are pursuing a degree in engineering or related fields.
Software Life Cycle: Development phase The phase in the software life cycle where software is designed, implemented, and tested.
Requirements analysis The process of gathering and analyzing the requirements for a software system.
Design The process of creating a plan for the software system.
Implementation The process of writing the code for the software system.
Testing The process of evaluating the software system to ensure that it meets the requirements and is free of defects
Modular implementation The approach of breaking down a software system into smaller modules or components, making it easier to develop and maintain.
Unified Modeling Language (UML) A standardized modeling language used in software engineering to visualize, design, and document software systems.
Use case A technique used in software engineering to identify, define, and organize requirements of a system or application.
Use case diagram A graphical representation of the use cases of a system or application.
Data relationships: one-to-one A relationship between two entities where one instance of an entity is associated with only one instance of another entity.
Data relationships: one-to-many A relationship between two entities where one instance of an entity is associated with multiple instances of another entity.
Data relationships: many-to-many A relationship between two entities where multiple instances of an entity are associated with multiple instances of another entity.
Design pattern A reusable solution to a commonly occurring problem in software design.
Software quality assurance (SQA) The process of ensuring that a software system meets the specified requirements and quality standards.
Software testing: glass box testing A testing technique that examines the internal workings of a software system or application.
Software testing: black box testing A testing technique that examines the external behavior of a software system or application without knowing its internal workings.
Software testing: beta testing A type of testing where the software system or application is tested by a group of end-users before its release.
Software testing: alpha testing A type of testing where the software system or application is tested in a controlled environment by the developers or testers before its release.
Documentation: user documentation The documentation that provides information to the end-users on how to use a software system or application.
Documentation: system documentation The documentation that provides information on the internal workings of a software system or application.
Documentation: technical documentation The documentation that provides technical details on how to install, configure, and maintain a software system or application.
Intellectual property law The law that governs the legal rights of creators and owners of intellectual property, including patents, trademarks, and copyrights.
Software license A legal agreement that outlines the terms and conditions for the use of a software system or application.
Software disclaimer A statement that limits the liability of the software developer or vendor for any damages or losses resulting from the use of the software system or application.
Best defenses against software liability claims The best defense agains software liability claims is to use a level of care compatible witht he software's application and to maintain records that verify the amount of care taken.
